마케팅 자동화 툴 프로젝트 · 회의 가이드

5일 스프린트 · 수요일 킥오프

한 상품의 마케팅 전략을 정할 때 필요한 결정들을, 팀원과 하나씩 회의로 정해 나가기 위한 체크리스트입니다.

📅 일정: 이번 주 수요일부터 총 5일. 오늘(화)에는 국가·제품군이 정해지고, 그걸 바탕으로 툴 기획·개발이 진행됩니다.

1. 팀 정체성 & 오늘 확정될 전제

마케팅 회사에서 프로젝트를 시작할 때 가장 먼저 정하는 것들입니다.

  • 팀명: 우리 팀을 부를 이름 하나 정하기
  • 타겟 국가: 오늘 확정 예정 — 정해지면 바로 툴 기획에 반영
  • 제품군: 오늘 확정 예정 — 가격·단가·광고 콘셉트의 기준이 됨
  • 툴의 목적 한 줄: “우리 툴은 ○○ 국가의 △△ 제품 마케팅을 위해 가격·광고·시장조사를 한곳에서 관리/자동화한다” 수준으로 공유

2. 한 상품 마케팅 전략에서 꼭 정하는 것들

실무에서 한 상품의 마케팅 전략을 세울 때 결정하는 항목들입니다. 우리 툴에 어떤 기능을 넣을지 여기서 골라서 범위를 정하면 됩니다.

  • 타겟 시장·고객: 누구를 대상으로 할지 (B2B / B2C / 연령·지역 등)
  • 가격 전략: 제시 가격, 할인 구조, 통화(원화/달러 등)
  • 단가·원가: 제품 단가, 마진, 국가별/채널별 단가 비교
  • 광고 콘텐츠: 영상 광고, 포스터/이미지 광고 — 톤앤매너, 메시지, 규격
  • 시장 조사: 경쟁사, 시장 규모, 트렌드 — 툴에서 어떤 지표를 보여줄지
  • 랜딩/웹페이지: 상품 소개 페이지, CTA(구매·문의 등) 구조
  • 채널 전략: SNS, 검색, 디스플레이 등 어떤 채널을 툴에서 다룰지

3. 툴 구조 (화면 구성 방식)

5일 안에 나눠 개발하려면 “페이지를 나눌지, 한 화면에 모을지”를 먼저 정하는 게 좋습니다.

추천 · 5일 일정에 유리

페이지 분리형 (기능별 페이지)

가격/단가, 광고(영상·포스터), 시장조사, 웹페이지 등 기능별로 HTML 페이지를 나누고 메뉴로 연결.

  • 역할 나누기 쉽고 작업 충돌 적음
  • 나중에 합치기 단순함
일정 타이트할 때 부담

단일 페이지 통합형 (All-in-One 대시보드)

한 화면에서 가격·광고·시장조사 등이 모두 연동되어 보이는 방식.

  • 완성도는 높지만 설계·통합 부담 큼
  • 데이터 규격·상태 관리 공유 필수

4. 역할 분담 방식

기능별로 쪼개서 담당할지, 2인 1조로 큰 카테고리씩 맡을지 정합니다.

방식 A. 기능별 역할 분담

가격/단가 1명, 광고(영상·포스터) 1명, 시장조사 1명, 웹페이지 1명 식으로 기능 단위로 나눔.

  • 담당 영역이 명확함
  • 인원이 4명 이상일 때 적합

방식 B. 2인 1조 · 카테고리 단위

예: 1조 = 가격/단가+시장조사, 2조 = 광고+랜딩페이지 — 조 단위로 큰 블록을 나눔.

  • 서로 백업·리뷰하기 좋음
  • 인원이 4명일 때 2조로 나누기 적당

역할–담당 매핑 (정한 방식에 맞춰 이름만 채우기)

역할/기능 주요 담당 내용 담당자
PM/통합 기획, 공통 데이터 규격(JSON 등), 코드 통합, 일정 관리
가격·단가 제시 가격, 단가, 국가별 비교, 차트/테이블
광고 (영상·포스터) 광고 콘텐츠 기획, 영상/포스터 생성·관리 UI
시장 조사 시장/경쟁사 데이터 구조, 시각화(표·차트)
웹페이지/랜딩 상품 소개 페이지, CTA, 공통 레이아웃·스타일
데이터/공통 국가·제품 정보 JSON/DB 구조, 다른 파트 보조

5. 기술 스택 & 공통 규칙

각자 다른 AI/도구를 써도 되지만, 합칠 때를 위해 공통 규격만 미리 정해 두면 5일 차에 덜 헷갈립니다.

항목 회의에서 정할 내용 예시
CSS 디자인 통일을 위해 어떤 방식 쓸지 Pure CSS / Tailwind / Bootstrap 중 하나
JS 수준과 범위 Vanilla JS 등
데이터 저장 가격·제품·국가 정보를 어디에 둘지 단일 JSON 파일 / LocalStorage 등
파일/폴더 규칙 페이지 파일명, 공통 이미지/폰트 경로 예: price.html, ads.html, research.html

6. 브랜딩 & 톤앤매너

툴 UI와 광고 콘텐츠에 공통으로 쓸 기준입니다.

  • 테마: 다크 모드 vs 라이트 모드 (또는 툴만 라이트, 광고는 별도)
  • 메인 컬러: 팀/상품을 대표하는 색 하나 (예: 브랜드 블루, 그린)
  • 타겟 국가 문화: 할당 국가의 색·이미지 선호(금기 등) 반영 여부
한마디: 수요일에 국가·제품이 나오면 그날 바로 설계도에 넣고 개발에 들어가면 됩니다. 오늘 회의에서는 “팀명, 역할 방식, 툴에 넣을 기능 범위, 공통 규격”만 확정해 두어도 다음 날부터 진행이 훨씬 수월합니다.

7. 우리가 가져갈 5일 계획표 (로드맵)

데이터 전처리 → 개별 빌드 → 통합 → 영상·발표 흐름을 유지하면서, 마케팅 자동화 특성(시각적 임팩트, 국가별 맞춤)에 맞춘 일정입니다. HTML/JS/CSS 사용 시 통합 단계를 명확히 두는 것이 중요합니다.

📌 수요일 회의 제안 멘트: "저번 ERP 때처럼 효율적으로 가되, 이번엔 한 화면에서 모든 게 돌아가는 '자동화 툴'을 만들자."

💡 이번 프로젝트에서 업그레이드할 'PM의 한 수' (3가지)

  • Streamlit → HTML 통합: 통합이 조금 더 정교해지므로, PM이 Cursor로 전체 대시보드 심미적 완성도를 챙기면 훨씬 '진짜 서비스'처럼 보입니다.
  • 데이터의 동적 활용: CSV만 읽는 수준을 넘어, 사용자 입력값에 따라 AI가 마케팅 포스터 문구를 실시간으로 바꿔 주는 기능을 넣으면 발표 반응이 좋습니다.
  • 영상 제작 시점: 코드가 다 합쳐지기 전이라도, 각 팀원의 개별 모듈 작동 영상을 미리 찍어 두세요. 나중에 통합본과 합치면 풍성한 발표 자료가 됩니다.

🚀 프로젝트명: AI 기반 글로벌 마케팅 자동화 시스템

1. 프로젝트 개요

  • 목표: 특정 국가/제품군 할당 시, 시장 분석 → 콘텐츠 제작 → 비즈니스 로직까지 한 번에 수행하는 원페이지 마케팅 대시보드 구축.
  • 핵심 컨셉: "데이터로 설득하고, AI로 시각화하며, 시스템으로 자동화한다."

2. 팀 구성 및 R&R (6인)

역할담당
PM (통합·관리)시스템 아키텍처 설계, 모듈 통합(Cursor), 팀 페이지·일정 관리
광고/콘텐츠 (2인)AI 홍보 영상, 노래(Suno), 광고 포스터 제작 및 UI
프로세스 (2명)시장 데이터 API 연동, 관세/환율 계산, 자동 계약서 생성 엔진
검증/인텔리전스 (1명)Perplexity 시장 조사, 데이터 팩트체크, 신뢰도 분석 리포트

3. 개발 로드맵 (요약)

§7 5일 계획표와 동일. Day 1 = Strategy(전략·JSON 규격) → Day 2–3 = Development(팀별 모듈) → Day 4 = Integration(Cursor 통합·영상) → Day 5 = Showcase(시연·발표).

4. 기술 스택 및 협업 규칙

  • Data Single Source: 모든 모듈은 shared_data.json 규격 준수.
  • CSS 접두사: 스타일 충돌 방지용 팀별 접두사 (예: .ad-, .pro-, .chk-).
  • Modular Design: 각 파트는 독립 폴더에서 index.html로 구동 가능하게, 핵심 기능은 함수 단위 모듈화.
  • AI 도구: Perplexity(시장 조사), Suno/Udio(오디오), ClovaNote(회의 기록) 등 팀 합의대로.

5. 최종 결과물 구성 (원페이지 대시보드)

  • Header: 국가/제품 선택, 프로젝트 로고.
  • Visual (좌): 생성형 AI 미디어 — 영상/노래/포스터.
  • Logic (우): 비즈니스 계산기, 문서 생성기.
  • Evidence (하단): 데이터 출처·신뢰도 검증 리포트(Perplexity 기반).
